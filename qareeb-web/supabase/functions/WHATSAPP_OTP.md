# تحقّق الكابتن عبر واتساب (OTP)

تدفّق تسجيل الكابتن: **الاسم + كلمة السر → رمز واتساب (OTP) → البصمة (اختياري) →
إنشاء الحساب → المستندات → اعتماد الأدمن → العمل**.

يعتمد على دالتَي Edge:

| الدالة | الوظيفة |
|--------|---------|
| `send-otp`   | يولّد رمزاً من 6 أرقام، يخزّن **تجزئته** في `otp_codes`، ويرسله عبر WhatsApp Cloud API. |
| `verify-otp` | يتحقّق من الرمز (انتهاء + محاولات ≤5) ثم يحذفه. |

## 1) قاعدة البيانات

نفّذ `supabase/schema.sql` المحدّث (ينشئ جدول `otp_codes` بلا سياسات RLS —
لا وصول من العميل؛ الدوال فقط تصل إليه بمفتاح service_role).

## 2) وضع التطوير (بلا إعداد واتساب)

إن لم تُضبط أسرار واتساب، تعمل الدالة في **وضع تجريبي**: تُعيد `devCode` في الرد،
ويظهر في شاشة الكابتن، فيكتمل التدفّق فوراً للاختبار. في وضع المعاينة (بلا Supabase)
الرمز الثابت هو `000000`.

## 3) الإنتاج — WhatsApp Cloud API (Meta)

1. أنشئ تطبيقاً في [Meta for Developers](https://developers.facebook.com/) وأضِف
   منتج **WhatsApp**، واربط رقم مُرسِل واحصل على **Phone Number ID** و**رمز وصول دائم**.
2. أنشئ **قالب مصادقة** (Authentication template) معتمداً — اسمه مثلاً `otp` —
   بمُعامل رمز في الجسم وزر «نسخ الرمز».
3. اضبط الأسرار (لا تُخزَّن في المستودع إطلاقاً):

```bash
supabase secrets set \
  WHATSAPP_TOKEN=EAAG... \
  WHATSAPP_PHONE_ID=123456789012345 \
  WHATSAPP_TEMPLATE=otp \
  WHATSAPP_LANG=ar \
  OTP_PEPPER=$(openssl rand -hex 16)
```

4. انشر الدالتين:

```bash
supabase functions deploy send-otp
supabase functions deploy verify-otp
```

> `SUPABASE_URL` و`SUPABASE_SERVICE_ROLE_KEY` يحقنهما Supabase تلقائياً داخل الدوال —
> لا تضِفهما يدوياً ولا تشاركهما مع العميل.

## ملاحظات أمان

- يُخزَّن **تجزئة** الرمز فقط (SHA-256 + pepper)، لا الرمز الصريح.
- حدّ إرسال: مرّة كل 60 ثانية للرقم. صلاحية الرمز 5 دقائق. 5 محاولات كحدّ أقصى.
- استدعاء الدوال بمفتاح anon عادي (قبل الدخول) آمن — لا تكشف الدوال الرمز.
