# قريب · Qareeb (web)

تطبيق نقل/توصيل سوداني — **نقل آمن في كل السودان**. هذه نسخة الويب (عميل/سائق/أدمن)
المبنية بـ **Vite + React + TypeScript + Tailwind + Supabase + خرائط قوقل**، بهوية
"قريب" الكاملة (RTL، خط Tajawal، الأخضر السوداني والذهبي).

## المكدّس (Stack)

- **Vite + React 18 + TypeScript**
- **Tailwind CSS** — الهوية معرّفة في `tailwind.config.ts`
- **Supabase** — Auth + Postgres + Storage + Realtime (`src/lib/supabase.ts`)
- **@react-google-maps/api** — الخرائط (`src/components/MapView.tsx`)
- **react-router-dom** — المسارات (`src/App.tsx`)

## التشغيل محلياً

```bash
cp .env.example .env      # املأ مفاتيح Supabase و Google Maps
npm install
npm run dev               # http://localhost:5173
```

> التطبيق يعمل حتى بدون مفاتيح: الخريطة والـ backend يعرضان بدائل واضحة
> بدل أن يتعطّل، لتسهيل معاينة الواجهة.

## البناء

```bash
npm run build             # ينتج dist/
npm run preview           # معاينة نسخة الإنتاج
```

## قاعدة البيانات

شغّل `supabase/schema.sql` في **Supabase → SQL Editor** (أو `supabase db push`).
يُنشئ الجداول: `users, drivers, rides, wallets, transactions, topups, settings`
مع RLS، ومحفظة تلقائية لكل مستخدم جديد، وصف إعدادات واحد للعمولة والحساب البنكي.

## النشر على Cloudflare Pages

> دليل مفصّل خطوة‑بخطوة (Supabase + Google Maps + Cloudflare): **[`DEPLOY.md`](./DEPLOY.md)**

1. اربط المستودع بـ Cloudflare Pages.
2. **Build command:** `npm run build`
3. **Build output directory:** `dist`
4. **Root directory:** `qareeb-web`
5. أضف متغيّرات البيئة (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`,
   `VITE_GOOGLE_MAPS_API_KEY`) في إعدادات المشروع.

توجيه الـ SPA مضبوط عبر `public/_redirects`.

## التسعير

الأجرة تُحسب في `src/lib/pricing.ts` باستراتيجية الشرائح:
`فتح العداد + (كم حضري × سعره) + (كم بعيد × سعره) + (دقائق × سعر الدقيقة) × Surge`.
المسافة والزمن من **Google Directions API** (`fetchRoute`)، مع بديل تقديري
(Haversine) عند غياب المفتاح. تسعيرة كل نوع مركبة في جدول `service_pricing`
وتُدار من لوحة الأدمن (مع Surge والعمولة وحدود الشرائح في `settings`).

## الوقت الفعلي (Realtime)

عبر Supabase Realtime على جدول `rides` (`src/lib/realtime.ts`):

- **السائق** يستقبل الطلبات الجديدة فور إنشائها (بدل الاستطلاع).
- **العميل** ينتقل لشاشة الرحلة لحظة قبول السائق، ولشاشة التقييم لحظة إنهائه.
- **ترحيل**: المنظّم يرى الركّاب ينضمّون لحظياً، والسائق يستقبل الطلبات المُرسَلة فوراً.
- بديل تلقائي في وضع المعاينة. النشر مُفعّل في `schema.sql` (rides + commute).

## دورة الرحلة والمال (اكتملت)

- **الدفع والتسوية موحّدان** في دالة `settle_ride` (يستدعيها السائق/الأدمن):
  - **محفظة قريب**: تُخصم الأجرة من محفظة العميل، ويُقيَّد للسائق (الأجرة − العمولة).
  - **كاش/تحويل**: يستلم السائق الأجرة مباشرة، وتُخصم العمولة فقط من محفظته.
  - العميل يقيّم فقط (`rateRide`)؛ الإنهاء والتسوية بيد السائق (لا اكتمال دون دفع).
- **السائق الحقيقي** يُعرض في شاشتي الرحلة والتقييم (اسم، تقييم، لوحة، اتصال) عبر
  دالة آمنة `get_ride_driver` بدل بيانات ثابتة.
- **قبول الترحيل**: زر السائق يعيّن الطلب له وينقله `dispatched → active`
  (`acceptCommuteOrder` + عمود `driver_id` + سياسة RLS).
- **استرجاع الرحلة الجارية** بعد تحديث الصفحة للعميل والسائق
  (`getActiveCustomerRide` / `getActiveDriverRide`).

## الانضمام كسائق (اكتمل)

نظام تسجيل سائق مستقل بالكامل — **لا رابط له من صفحات العميل**. مدخله من شاشة
الدخول («الدخول أو الانضمام كسائق»)، وحارس `/driver` يوجّه غير السائقين إلى
`/driver/register`.

- **صفحة `/driver/register`** تجمع: الاسم، الهاتف، البريد، نوع السيارة (من الكتالوج)،
  لوحة السيارة، السكن، وهل هي مستأجرة — مع رفع الوثائق والصور:
  رخصة القيادة، رخصة السيارة، تصريح النقل، عقد الإيجار (إن كانت مستأجرة)،
  وصور السيارة (أمامية/خلفية/جانبية/داخلية) إلى bucket خاص `driver-docs`.
- **سير الاعتماد**: يُرسَل الطلب `pending` → يعتمده الأدمن → عندها فقط يصبح الدور
  `driver` ويُنشأ صف `drivers` (عبر `approve_driver_application`). الرفض يعرض السبب
  مع خيار إعادة التقديم.
- **لوحة الأدمن**: قائمة مستقلة لطلبات الانضمام مع معاينة كل وثيقة/صورة واعتماد/رفض.
- **السائق يطلب مشواراً أو مساعدة** (عند التعطّل) من واجهته كأي راكب.

جدول `driver_applications` + دوال الاعتماد/الرفض + سياسات RLS و bucket الوثائق في
`supabase/schema.sql`.

## تقدّم الرحلة والترحيل (اكتمل)

- **مسار حالة الرحلة الكامل**: السائق يعلّم *وصلت لموقع الراكب* (`arrived`) ثم
  *بدء الرحلة* (`in_progress`) ثم *إنهاء وتسوية* — عبر دالة آمنة `set_ride_status`
  (الإكمال يبقى حصراً في `settle_ride` لضمان الدفع). الراكب يرى لافتة حالة حيّة
  (في الطريق إليك / وصل السائق / الرحلة جارية) عبر Realtime.
- **رحلات الترحيل المقبولة لدى السائق**: واجهة الترحيل تعرض قسمين — *رحلاتك
  المقبولة* (`active`) و*طلبات متاحة* (`dispatched`) — عبر `listDriverCommutes`.

## الفصل بين التطبيقات (اكتمل)

أقسام مستقلة بنشر واحد: العميل (`/`)، السائق (`/driver/login`)، الإدارة
(`/admin/login`) — لكل منها دخول وهوية وتنقّل خاص، بلا روابط متبادلة.

## لوحة الأدمن (اكتملت — تبويبات)

- **نظرة عامة**: إحصاءات + ملخّص مالي (عمولة المنصة، أرباح السائقين، التعبئات،
  مدفوعات المحفظة، الرحلات المكتملة، أرصدة المحافظ) عبر `admin_financial_summary`.
- **الطلبات**: اعتماد التعبئات + مراجعة طلبات الانضمام كسائق ووثائقها.
- **السائقون**: كل السائقين (حالة الاتصال، المركبة، اللوحة، التقييم).
- **الرحلات**: أحدث الرحلات وحالاتها وأجورها.
- **الإعدادات**: تسعير المركبات + العمولة/Surge/الشرائح + الحساب البنكي.

صلاحيات القراءة للأدمن على `drivers/rides/transactions` مضبوطة بـ RLS.

## إشعارات Web Push (اكتملت — تتطلّب نشراً)

إشعارات خلفية للسائق (طلب جديد) والراكب (تغيّر حالة الرحلة) واعتماد السائق
والترحيل — عبر Service Worker + جدول `push_subscriptions` + دالة Edge `push`.
خطوات الإعداد والنشر ومفاتيح VAPID في **[`supabase/NOTIFICATIONS.md`](./supabase/NOTIFICATIONS.md)**
(والمهام المؤجّلة في **[`PENDING.md`](./PENDING.md)**).

## تتبّع موقع السائق لحظياً (اكتمل)

يبثّ السائق موقعه أثناء الرحلة عبر قناة **Broadcast** (بلا كتابة في القاعدة)،
والراكب يراه نقطةً خضراء تتحرّك على الخريطة وتتبعه المركزية. `createLocationBroadcaster`
/ `subscribeToDriverLocation` في `src/lib/realtime.ts`.

## إلغاء الرحلة (اكتمل)

عبر دالة آمنة `cancel_ride`:
- **الراكب** يلغي (بحث/قبل بدء الرحلة) → `cancelled`؛ يُبلَّغ السائق لحظياً.
- **السائق** يتخلّى عن رحلة قبِلها → تعود `searching` بلا سائق (تُتاح لغيره)،
  ويُعاد الراكب لشاشة البحث لحظياً.
