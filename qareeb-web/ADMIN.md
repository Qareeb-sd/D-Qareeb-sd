# موقع لوحة الإدارة «قريب»

لوحة الإدارة **موقع ويب فقط** (ليست تطبيقاً)، مستقلّة عن تطبيقَي العميل والسائق —
لا تُحمَّل داخل أي APK. منها تُدير المنصّة: **اعتماد/رفض طلبات السائقين**، طلبات
تعبئة المحفظة، التسعير، العمولة، الحساب البنكي، الملخّص المالي، وتنبيهات الطوارئ.

## التشغيل محلياً (الأسرع للاستخدام الآن)
من مجلد `qareeb-web` (مع وجود `.env` بمفاتيح Supabase):
```
npm run dev:admin
```
ثم افتح الرابط الظاهر (مثل `http://localhost:5173`) في المتصفح.

## البناء للنشر (موقع دائم)
```
npm run build:admin      # يُنتج dist/ جاهزاً للنشر
```
ارفع مجلد `dist/` إلى أي استضافة ثابتة (Netlify / Vercel / Cloudflare Pages…).
ملف `public/_redirects` يضبط توجيه صفحات SPA على Netlify.

> انشره على **نطاق/نطاق فرعي منفصل** (مثل `admin.qareeb.sd`) — ليس نفس موقع العملاء.

## الدخول
- الرابط الجذر للموقع = **شاشة دخول الإدارة**.
- سجّل دخول برقم هاتف **دوره `admin`** في قاعدة البيانات + كلمة السر.
- من ليس أدمن يُمنع (الحماية مفروضة أيضاً عبر RLS ودوال Postgres، لا الواجهة فقط).

> إن رفض الدخول لحساب ضبطته عبر SQL: تأكّد أن الحساب **سجّل دخوله مرّة من التطبيق**
> (حتى يُنشأ في نظام مصادقة Supabase) ثم عُيِّن دوره `admin` على نفس الصف.

## الأقسام
| القسم | الوظيفة |
|---|---|
| نظرة عامة | رحلات اليوم · سائقون متصلون · تعبئات معلّقة · الملخّص المالي |
| الطلبات | **اعتماد/رفض السائقين** (مع عرض الوثائق والصور) + طلبات التعبئة |
| السائقون | كل السائقين وحالتهم |
| الرحلات | أحدث الرحلات وحالاتها |
| الإعدادات | التسعير لكل مركبة · العمولة · Surge · الحساب البنكي |
